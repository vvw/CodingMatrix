The field

Object-oriented conception

In linear algebra, data abstraction give us the concept of a field. Think in a field as a class that implements the operations plus, minus, times abd divide. As long as a class's implementation of these operations satisfies certain algebraic rules, we can use it to do linear algebra

Much of linear algebra is based on +, -, *, / and some algebra properties like:

  • / is inverse of *
  • - is inverse of +
  • addition is conmutative: \(a+b = b+a\)
  • multiplication distributes over addition: \(a*(b+c) = a*b + a*c\)

You can plug in any collection of "numbers" with arithmetic operators +, -, *, / satisfying the algebraic properties — and much of linear algebra will still "work".

Such a collection of "numbers" with +, -, *, / is called a field. Different fields are like different classes obeying the same interface.
